# Writeup for Leaked Flag Checker

## 問題

> みんなだいすきフラグチェッカー

C言語のソースコードと実行ファイルが配布される。
実行ファイルを起動し、フラグを一致させる必要がある…かと思えば、実行ファイルを覗くだけでOK。
ソースコードには`const char xor_flag[] = "REDACTED";`と書いているが、これが答えなわけではない。

## 解法

中の処理では、フラグに対して`^7`している。
7（0b00000111）とのXORなので、下位3ビットについて反転させている。
`Alpaca`という文字が入っていると予想されるため、これを下位3ビット反転させると、`Fkwfdf`となる。
バイナリエディターで実行ファイルを開き`Fkwfdf`を探すと、以下のバイナリが見つかる。
（改行は命令ごとに整理）

```asm
; mov rax, 0x6b7c666466776b46
48 b8 46 6b 77 66 64 66 7c 6b

; mov [rbp-0x3e], rax
48 89 45 c2

; mov rax, 0x007a7e6c64726b7c
48 b8 7c 6b 72 64 6c 7e 7a 00

; mov [rbp-0x38], rax
48 89 45 c8
```

`0x48, 0xb8`はx86-64の命令で、内容は「この直後の8バイトをレジスタに入れる」。
`0x48, 0x89`はmov命令で、`0x48, 0x89, 0x45, 0xc2`なら`rbp-0x3e`に書く、という意味になる。
（理由は["`0x48, 0x89`解説"](#0x48-0x89解説)参照）
1行目は`rbp-0x3e`から8バイト、2行目は`rbp-0x38`から8バイトとなり、0x3e - 0x38 = 0x06なので、6バイトずらして書き込まれる。
8バイト書き込みなので、2バイト分が重複する。
つまり、書き込む内容はこうなる。
`0x46, 0x6b, 0x77, 0x66, 0x64, 0x66, 0x7c, 0x6b, 0x72, 0x64, 0x6c, 0x7e, 0x7a, 0x00`

あとは、この数値列を下位3ビット反転させればOK。

```python
data = [0x46, 0x6b, 0x77, 0x66, 0x64, 0x66, 0x7c, 0x6b, 0x72, 0x64, 0x6c, 0x7e, 0x7a, 0x00]

output = ""
for b in data:
    output += chr(b ^ 7)

print(output.rstrip('\x00'))
```

Flag:`Alpaca{lucky}`

## `0x48, 0x89`解説

`0x48, 0x89, 0x45, 0xc2`がどうして`rbp-0x3e`になるかを解説する。
分解するとこうなる。

`0x48`:64bitモード
`0x89`:mov r/m64, r64
`0x45, 0xc2`:アドレス指定（rbp + disp8）
つまり意味は：
`mov [rbp + c2], rax`（計算用レジスタの中身をメモリに書き込む）

`c2`はsignedなので、`-62`となる。
以下の計算が早い。
0xc2 = 194
256 - 194 = 62
